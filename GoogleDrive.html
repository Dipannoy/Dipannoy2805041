Scalable File Management for LMS

In a modern Learning Management System (LMS), file exchange is a core utilityâ€”from question paper distribution to student answer script submissions. To ensure scalability and data integrity, I integrated Google Drive REST APIs with an ASP.NET Core backend.

Implementation Architecture

The system was designed to handle various file types and sizes efficiently, optimizing network bandwidth and ensuring successful transfers even over unstable connections.

  • Multipart Upload: Optimized for small metadata-heavy files (less than 5MB).
  • Resumable Upload: Implemented a chunk-based protocol for large files (greater than 5MB), allowing transfers to resume if interrupted.
  • Streamed Downloads: Implemented byte-array retrieval from the API, piping the data directly to the user's local stream to minimize server memory overhead.
  • CRUD Operations: Full lifecycle management including programmatic file deletion and permission handling via FileID.

Impact

By offloading storage to Google Drive, the system eliminated the need for expensive on-premise storage servers while providing enterprise-grade security and 99.9% availability for academic documents.
